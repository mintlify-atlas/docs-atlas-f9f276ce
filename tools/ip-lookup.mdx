## Overview

The IP Lookup tool performs geolocation queries on IPv4 and IPv6 addresses using the [ip.guide](https://ip.guide) API. It retrieves comprehensive information including geographic location, network details, autonomous system (AS) information, and ISP data.

<Note>
  The tool supports both IPv4 and IPv6 addresses with built-in validation using Django's `validate_ipv46_address` validator.
</Note>

## How It Works

The tool sends HTTP requests to a configurable IP geolocation API and parses the JSON response to extract location, network, and ASN information.

### Technical Implementation

From `IPLookup/views.py:23-40`:

```python
def ip_search(request):
    if request.method == "POST":
        form = IPLookupForm(request.POST)
        if form.is_valid():
            ip = form.cleaned_data["ip"]
            api_url = _build_api_url(ip)
            
            try:
                resp = httpx.get(api_url, timeout=15.0)
                try:
                    data = resp.json()
                except Exception:
                    data = {"raw": resp.text, "status_code": resp.status_code}
                
                request.session["iplookup_results"] = data
                request.session["iplookup_ip"] = ip
                return redirect(reverse("iplookup:results"))
```

## IP Address Validation

The tool validates IP addresses using Django's built-in validator:

From `IPLookup/forms.py:12-20`:

```python
def clean_ip(self):
    ip = self.cleaned_data.get("ip")
    try:
        validate_ipv46_address(ip)
    except forms.ValidationError:
        raise forms.ValidationError(
            "Introduce una dirección IP válida IPv4 o IPv6."
        )
    return ip
```

This ensures:
- Valid IPv4 format (e.g., `192.168.1.1`)
- Valid IPv6 format (e.g., `2001:db8::1`)
- Rejection of invalid strings or malformed addresses

## API Configuration

The tool uses a flexible API URL template system:

### URL Template Building

From `IPLookup/views.py:12-20`:

```python
def _build_api_url(ip: str) -> str:
    template = getattr(settings, "IP_GUIDE_API_TEMPLATE", "https://ip.guide/{ip}")
    api_key = getattr(settings, "IP_GUIDE_API_KEY", None)
    try:
        if "{api_key}" in template:
            return template.format(ip=ip, api_key=api_key or "")
        return template.format(ip=ip)
    except Exception:
        return template.replace("{ip}", ip)
```

### Configuration Options

<ParamField path="IP_GUIDE_API_TEMPLATE" type="string" default="https://ip.guide/{ip}">
  API endpoint template with `{ip}` placeholder for the IP address. Can include `{api_key}` placeholder for authenticated requests.
</ParamField>

<ParamField path="IP_GUIDE_API_KEY" type="string" default={null}>

## HTTP Client

The tool uses `httpx` for HTTP requests with:

- **Timeout**: 15 seconds
- **Error Handling**: Graceful degradation on API failures
- **JSON Parsing**: Automatic with text fallback

```python

try:
```

## Session Storage

Results are stored in Django sessions:

```python
request.session["iplookup_results"] = data

```

This allows:
- Persistent results across page navigation
- No database overhead
- Automatic cleanup on session expiration

### Reserved IP Ranges

```plaintext
127.0.0.1       → Loopback (local machine)
192.168.1.1     → Private network (RFC 1918)
10.0.0.1        → Private network (RFC 1918)
::1             → IPv6 loopback
```

<Note>
  Private IP addresses (RFC 1918) may return generic or no results as they are not routable on the public internet.
</Note>

## Limitations

- API rate limits may apply depending on the service
- Geolocation accuracy varies (typically city-level)
- Private IP addresses return no meaningful data
- VPN/proxy usage can obscure true location
- Database accuracy depends on RIR registration updates
- Some API providers require authentication for full features

## Alternative APIs

The tool's flexible configuration supports various IP geolocation APIs:

- **ip.guide**: Default, free, no registration required
- **ipinfo.io**: `https://ipinfo.io/{ip}/json?token={api_key}`
- **ipapi.co**: `https://ipapi.co/{ip}/json/`
- **ip-api.com**: `http://ip-api.com/json/{ip}`
- **ipgeolocation.io**: `https://api.ipgeolocation.io/ipgeo?apiKey={api_key}&ip={ip}`

Configure via `settings.py`:

```python
IP_GUIDE_API_TEMPLATE = "https://ipinfo.io/{ip}/json?token={api_key}"
IP_GUIDE_API_KEY = "your_api_key_here"
```
